# Skill 配置

在设置面板的 **Skill** 标签页中，你可以管理本地写作技能（Skill）：上传压缩包、启用或停用、删除不再使用的技能。

当 Skill 启用后，智能体会在对话过程中自动发现并按需加载对应规则，帮助你在特定写作场景下输出更稳定的内容。

![](/skill.png)

## Skill 是什么

Skill 本质上是一组本地 Markdown 规则文件，核心入口是 `SKILL.md`。

- `SKILL.md`：定义技能名称、描述和主规则
- 其他 `.md` 文件：可作为补充说明（方法库、示例、术语表等）

系统会自动扫描本地 `wence_data/skills` 目录中包含 `SKILL.md` 的文件夹，并将其显示在 Skill 列表中。

## 上传 Skill 压缩包

1. 打开设置面板，切换到 **Skill** 标签页
2. 点击 **上传 Skill 压缩包**
3. 选择本地 `.zip` 文件
4. 上传成功后，列表会自动刷新

### 压缩包要求

- 仅支持 `.zip` 格式
- 压缩包中必须包含且只包含 **一个** `SKILL.md`
- `SKILL.md` 文件名大小写不敏感（如 `skill.md` 也可识别）

如果不满足要求，会看到类似提示：

- `压缩包中未找到 SKILL.md，请上传包含 SKILL.md 的 skill 压缩包`
- `压缩包中包含多个 SKILL.md，请一次只上传一个 skill`

### 命名与覆盖规则

- 如果 `SKILL.md` 在某个子文件夹内：该文件夹名会作为 Skill 文件夹名
- 如果 `SKILL.md` 在压缩包根目录：使用 zip 文件名作为 Skill 文件夹名
- 如果本地已存在同名 Skill：新上传内容会覆盖旧内容

## 启用 / 停用 Skill

每个 Skill 右侧都有一个开关：

- **开启**：该 Skill 可被智能体使用
- **关闭**：该 Skill 保留在本地，但不会参与智能体能力构建

新上传的 Skill 默认是启用状态。

## 删除 Skill

点击某个 Skill 右侧的删除按钮即可移除。

- 删除后会清理本地 Skill 文件夹
- 同时清理该 Skill 的启用状态记录


## 何时会自动使用 Skill

当你的提问意图与某个 Skill 匹配时，智能体会优先读取该 Skill 的规则内容，并结合文档工具执行写作任务。

例如你可以这样提问：

- `按 writing-assistant 的规则给我出一版文章大纲`
- `使用 copywriting skill 帮我重写这段产品文案`

## 常见问题

### 上传成功但列表没有出现

- 确认压缩包内只有一个 `SKILL.md`
- 确认上传的是 `.zip`，不是 `.rar` 或 `.7z`
- 重新打开设置面板后再查看列表

### Skill 已上传但聊天时没有生效

- 检查该 Skill 开关是否处于启用状态
- 在提问中明确指出希望使用的 Skill 名称

### 为什么列表名称和文件夹名不一样

列表优先显示 `SKILL.md` frontmatter 里的 `name`；如果未配置，才显示文件夹名。
